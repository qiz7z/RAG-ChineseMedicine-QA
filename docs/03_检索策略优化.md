## 二、优化一：药品名去空格

### 2.1 问题

Word 文档中药品名含空格（如"人 参"），但用户查询为"人参"，导致：
- 查询解析器无法在查询中匹配到药品名
- 元数据过滤条件无法生效
- BM25 分词后"人"和"参"被分开，检索精度下降

### 2.2 修复

**文件**：`src/etl/cleaner.py` → `clean_entry` 函数

```python
drug_name = drug_name.replace(' ', '').replace('\u3000', '')
```

同时处理 `parent_drug` 字段，确保子剂型的父级药品名也去空格。

### 2.3 效果

93% 的检索失败源于此问题，修复后 Hit@5 从 55% 提升至 64%。

## 三、优化二：BM25 后过滤

### 3.1 问题

BM25 索引基于 jieba 分词 + BM25Okapi，不存储元数据。当识别到药品名时：
- 向量检索已通过 `where` 条件过滤 ✓
- BM25 检索返回全库结果，包含大量无关药品 ✗
- RRF 融合时，无关药品的 BM25 高分 chunk 稀释了正确结果

### 3.2 修复

**文件**：`src/retrieval/retriever.py`

对 BM25 结果增加基于 SQLite 元数据的后过滤：

```python
if self.enable_filter and parsed["drug_names"]:
    filter_drugs = parsed["drug_names"]
    if bm25_results:
        # 1. 批量查询 BM25 结果的元数据
        bm25_ids = [r["id"] for r in bm25_results]
        bm25_metas = self.meta_store.get_by_ids(bm25_ids)
        bm25_meta_map = {m["chunk_id"]: m for m in bm25_metas}
        
        # 2. 逐条检查是否匹配过滤药品
        filtered_bm25 = []
        for r in bm25_results:
            meta = bm25_meta_map.get(r["id"], {})
            drug = meta.get("drug_name", "")
            for filter_drug in filter_drugs:
                if filter_drug in drug or drug in filter_drug:
                    r["metadata"] = meta
                    filtered_bm25.append(r)
                    break
        bm25_results = filtered_bm25
```

### 3.3 子串匹配策略

过滤使用双向子串匹配：
- `filter_drug in drug`：如 "人参" 匹配 "人参-饮片"
- `drug in filter_drug`：如 "人参" 匹配 "人参叶"（当用户查询"人参叶"时）

### 3.4 召回量补偿

过滤会损失部分结果，因此增大 BM25 召回量：

```python
bm25_top_k = BM25_TOP_K * 4  # 15 * 4 = 60
```

### 3.5 效果

Hit@5 从 64% 提升至 76%。

## 四、优化三：多药品查询过滤

### 4.1 问题

**之前的多药品查询逻辑**：

```python
# 旧代码：多药品名时不在向量检索中过滤
chroma_filter = None
if self.enable_filter and parsed["filter"]:
    if "$in" not in parsed["filter"]:
        chroma_filter = parsed["filter"]
    # 多药品名时跳过过滤（让向量检索全库召回，由重排来筛选）
```

**问题**：
- 查询"丹参和川芎的活血作用比较"时，识别到 ["丹参", "川芎"] 两个药品名
- 向量检索不做过滤，全库召回 → 返回大量无关药品
- BM25 也不做过滤 → 更多噪声
- RRF 融合后被无关结果稀释
- 重排虽然能部分修正，但候选池中正确结果太少

### 4.2 修复

**文件**：`src/retrieval/retriever.py`

**向量检索**：对多药品查询也应用 `$in` 过滤：

```python
# 修复后：所有药品名查询都做过滤
chroma_filter = None
if self.enable_filter and parsed["drug_names"]:
    filter_drugs = parsed["drug_names"]
    # 从 SQLite 获取所有匹配的药品名（包括 "-饮片" 等变体）
    all_drug_names = self.meta_store.get_all_drug_names()
    expanded_drugs = set()
    for fd in filter_drugs:
        for dn in all_drug_names:
            if fd in dn or dn in fd:
                expanded_drugs.add(dn)
    if expanded_drugs:
        chroma_filter = {"drug_name": {"$in": list(expanded_drugs)}}
```

**BM25 检索**：扩展为多药品匹配：

```python
# 修复后：多药品也做过滤
if self.enable_filter and parsed["drug_names"]:
    filter_drugs = parsed["drug_names"]
    # ... 对每个 BM25 结果，检查是否匹配任一过滤药品
    for filter_drug in filter_drugs:
        if filter_drug in drug or drug in filter_drug:
            filtered_bm25.append(r)
            break
```

### 4.3 FAISS 过滤器扩展

FAISS 的 `where` 过滤做精确匹配，需要将 "人参" 扩展为 ["人参", "人参-饮片", "人参叶"] 等所有变体：

```python
# 从 SQLite 获取所有药品名
all_drug_names = self.meta_store.get_all_drug_names()
expanded_drugs = set()
for fd in filter_drugs:
    for dn in all_drug_names:
        if fd in dn or dn in fd:
            expanded_drugs.add(dn)
chroma_filter = {"drug_name": {"$in": list(expanded_drugs)}}
```

### 4.4 效果

跨品种比较查询 Hit@5 从部分命中提升到 **100%**。

## 五、过滤逻辑对比

### 5.1 修复前

| 查询类型 | 向量检索 | BM25 检索 |
|---------|---------|----------|
| 无药品名 | 不过滤 ✓ | 不过滤 ✓ |
| 单药品名 | 精确过滤 ✓ | 后过滤 ✓ |
| 多药品名 | **跳过过滤** ✗ | **不过滤** ✗ |

### 5.2 修复后

| 查询类型 | 向量检索 | BM25 检索 |
|---------|---------|----------|
| 无药品名 | 不过滤 ✓ | 不过滤 ✓ |
| 单药品名 | $in 过滤（含变体）✓ | 后过滤 ✓ |
| 多药品名 | $in 过滤（含变体）✓ | 后过滤（多药品匹配）✓ |

### 6.2 过滤条件生成

```python
def _build_filter(self, drug_names: List[str]) -> Optional[Dict]:
    if not drug_names:
        return None
    if len(drug_names) == 1:
        return {"drug_name": drug_names[0]}
    else:
        return {"drug_name": {"$in": drug_names}}
```
